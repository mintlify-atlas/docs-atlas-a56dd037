## Overview

Creates a service creator that provides wallet connection functionality through `@hot-labs/near-connect`. This service creator is used by `createNearStore()` to enable wallet connect/disconnect operations and transaction signing.

## Function Signature

```typescript
function createNearConnectorService(
  args: { networkId: NearConnectNetworkId }
): NearConnectServiceCreator
```

### Type Definitions

```typescript
type NearConnectNetworkId = 'mainnet' | 'testnet';

type NearConnectServiceCreator = ServiceCreator<
  'nearConnector',
  { connector: NearConnector }
>;

type ServiceCreator<ServiceId, ServiceBox> = {
  serviceId: ServiceId;
  createService(): Service<ServiceId, ServiceBox>;
  createSigner(args: {
    signerAccountId: AccountId;
    serviceBox: ServiceBox;
    client: Client;
  }): Signer<ServiceId>;
};

type Service<ServiceId, ServiceBox> = {
  serviceId: ServiceId;
  serviceBox: ServiceBox;
};

type Signer<ServiceId> = {
  serviceId: ServiceId;
  safeExecuteTransaction(args: {
    intent: TransactionIntent;
  }): Promise<Result<ExecuteTransactionOutput, unknown>>;
};
```

## Parameters

<ParamField path="networkId" type="'mainnet' | 'testnet'" required>
  The NEAR network identifier. Must be either `'mainnet'` for production or `'testnet'` for testing. This determines which network the wallet connector will use.
</ParamField>

## Usage

### Mainnet Configuration

```tsx
import {
  createNearStore,
  createMainnetClient,
  createNearConnectorService,
} from 'react-near-ts';

const nearStore = createNearStore({
  networkId: 'mainnet',
  clientCreator: createMainnetClient,
  serviceCreator: createNearConnectorService({ networkId: 'mainnet' }),
});
```

### Custom Client with Service Creator

```tsx

  createNearStore,

const clientCreator = () => createClient({
```

## Implementation Details

The service creator follows a two-phase initialization pattern:

1. **Service Creation** (`createService`) - Instantiates the `NearConnector` with the specified network. This happens once when the store is created.

2. **Signer Creation** (`createSigner`) - Creates a signer instance when a wallet connects. The signer provides the `safeExecuteTransaction` method that wraps transaction execution with error handling.

The service creator is designed to be extensible, allowing for future support of multiple wallet connection services in the same store.

## Service Architecture

The service follows this structure:

```typescript
{
  serviceId: 'nearConnector',
  createService: () => {
    const connector = new NearConnector({ network: networkId });
    return {
      serviceId: 'nearConnector',
      serviceBox: { connector },
    };
  },
  createSigner: ({ serviceBox }) => {
    return {
      serviceId: 'nearConnector',
      safeExecuteTransaction: createSafeExecuteTransaction(serviceBox.connector),
    };
  },
}
```

The `serviceBox` acts as a container for the connector instance, making it accessible to both the service and signer contexts.

## Transaction Execution

The signer's `safeExecuteTransaction` method:

- Accepts a `TransactionIntent` with the action and receiver
- Uses the connected wallet to sign and submit the transaction
- Returns a `Result` type that safely encapsulates success or failure
- Provides raw RPC result on success for further processing
