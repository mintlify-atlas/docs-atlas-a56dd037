---

`TestnetNearProvider` is a convenience wrapper around `NearProvider` that automatically configures your application for the NEAR testnet network.

## Component Signature

```tsx
const TestnetNearProvider: (props: { children: ReactNode }) => JSX.Element
```

## Props

<ParamField path="children" type="ReactNode" required>
  The child components that will have access to the NEAR testnet context.
</ParamField>

## Usage

### Basic Setup

Wrap your application with `TestnetNearProvider` to connect to NEAR testnet:

```tsx
import { TestnetNearProvider } from 'react-near-ts';

export const App = () => (
  <TestnetNearProvider>
    <h1>Hello, Near!</h1>
  </TestnetNearProvider>
);
```

### With Router (Next.js App Router)

```tsx
import { TestnetNearProvider } from 'react-near-ts';

export default function RootLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  return (
    <html lang="en">
      <body>
        <TestnetNearProvider>
          {children}
        </TestnetNearProvider>
      </body>
    </html>
  );
}
```

### Development Workflow

`TestnetNearProvider` is ideal for development and testing:

```tsx
import { TestnetNearProvider } from 'react-near-ts';
import { useNearConnect, useNearViewCall } from 'react-near-ts';

function App() {
  return (
    <TestnetNearProvider>
      <TestComponent />
    </TestnetNearProvider>
  );
}

function TestComponent() {
  const { connect, connectedAccountId } = useNearConnect();
  const { data } = useNearViewCall({
    contractId: 'hello.test.near',
    method: 'get_greeting',
    args: {},
  });

  return (
    <div>
      {connectedAccountId ? (
        <p>Connected as: {connectedAccountId}</p>
      ) : (
        <button onClick={connect}>Connect to Testnet</button>
      )}
      <p>Greeting: {data}</p>
    </div>
  );
}
```

## Implementation Details

`TestnetNearProvider` creates a NEAR store configured for testnet with:

- **Network ID**: `testnet`
- **Client**: Testnet RPC client from `near-api-ts`
- **Services**: NEAR Connector service for testnet

<Info>
  Use this provider during development and testing. Switch to `MainnetNearProvider` for production deployments.
</Info>

<Note>
  Testnet accounts and tokens have no real value. You can create testnet accounts and request test NEAR tokens from the [NEAR testnet faucet](https://near-faucet.io/).
</Note>
