## Overview

`useNearConnector` provides wallet connection functionality for NEAR Protocol. It returns mutation functions to connect and disconnect wallets, managing the connection state and signers in the application.

## Import

```typescript
import { useNearConnector } from 'react-near-ts';
```

## Type Signature

```typescript

  disconnect: UseMutationResult<void, Error, void, unknown>;
};
```

## Usage

### Basic Wallet Connection

```typescript
import { useNearConnector } from 'react-near-ts';

function WalletButton() {
  const { connect, disconnect } = useNearConnector();

  return (
    <div>
      <button onClick={() => connect.mutate()}>
        Connect Wallet
      </button>
      <button onClick={() => disconnect.mutate()}>
        Disconnect
      </button>
    </div>
  );
}
```

### With Loading States

```typescript
import { useNearConnector, useConnectedAccount } from 'react-near-ts';

function ConnectButton() {
  const { connect, disconnect } = useNearConnector();
  const { isConnectedAccount, connectedAccountId } = useConnectedAccount();

  if (isConnectedAccount) {
    return (
      <button 
        onClick={() => disconnect.mutate()}
        disabled={disconnect.isPending}
      >
        {disconnect.isPending ? 'Disconnecting...' : `Disconnect ${connectedAccountId}`}
      </button>
    );
  }

  return (
    <button 
      onClick={() => connect.mutate()}
      disabled={connect.isPending}
    >
      {connect.isPending ? 'Connecting...' : 'Connect Wallet'}
    </button>
  );
}
```

### Error Handling

```typescript
import { useNearConnector } from 'react-near-ts';

function WalletConnection() {
  const { connect, disconnect } = useNearConnector();

  return (
    <div>
      <button onClick={() => connect.mutate()}>Connect</button>
      
      {connect.isError && (
        <p>Connection failed: {connect.error.message}</p>
      )}
      
      {disconnect.isError && (
        <p>Disconnection failed: {disconnect.error.message}</p>
      )}
    </div>
  );
}
```

## Behavior

### Connection Flow

1. Calls `connector.connect()` to open wallet selector
2. Retrieves connected accounts from the wallet
3. Sets the first account as the connected account
4. Initializes signers for the connected account
5. Updates application state with connection info

### Disconnection Flow

1. Calls `connector.disconnect()` to close wallet connection
2. Clears all signers from state
3. Removes the connected account ID
4. Returns application to disconnected state
